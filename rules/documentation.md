# Documentation Standards

Thin pointer — progressive documentation is part of every stage; the authoritative
guidance lives in `AGENTS.md` and the `ship`/`verify` skills. Do not duplicate it here.

Update documentation as you go, not at the end:

- **/plan** — record design intent, rationale, and alternatives in the plan doc.
- **/dev** — comment the *why* for non-obvious logic; keep JSDoc/TSDoc current.
- **/ship** — update README/CHANGELOG for user-facing or breaking changes.
- **/verify** — cross-check docs, validate examples, fix broken internal links.

Keep terminology consistent, link to code, and remove stale docs promptly.
